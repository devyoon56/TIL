# 소멸자

소멸자는 인스턴스가 메모리에서 해제되기 직전에 정리가 필요한 내용을 구현하는 메서드입니다.

## 소멸자의 특징

- 클래스를 정의할 때 클래스에 최대 1개의 소멸자를 정의할 수 있습니다.
- 소멸자는 클래스에만 존재하는 메서드입니다.
- 소멸자는 파라미터를 사용하지 않습니다.
- 소멸자를 직접 호출할 수 없습니다. (소멸자를 호출하는 문법이 없음)
- 소멸자는 클래스의 인스턴스가 메모리에서 해제되기 직전에 자동으로 호출됩니다.

## 소멸자의 작동 방식

- 스위프트는 클래스의 인스턴스를 ARC(자동 참조 계산) 방식을 통해 메모리에서 관리합니다.
- 강한 순환 참조를 제외한 일반적인 경우 인스턴스가 메모리에서 해제될 때 따로 관리할 필요는 없습니다.
- 그러나 인스턴스가 어떤 특별한 작업을 수행중인 경우 인스턴스를 메모리에서 해제하기 전에 몇 가지 추가적인 정리를 직접 수행할 수 있습니다.
  - 예시: 인스턴스에서 어떤 파일을 열고 파일에 데이터를 쓰는 경우 클래스의 인스턴스가 메모리에서 해제되기 전에 열었던 파일을 닫아야 파일에 손상을 입히지 않습니다.
  - 소멸자에서 인스턴스의 모든 속성에 접근할 수 있기 때문에 해당 속성에 접근하여 원하는 동작을 수행할 수 있습니다.

## 상속이 있는 경우

- 슈퍼 클래스의 소멸자는 서브 클래스에 자동 상속됩니다.
- 슈퍼 클래스의 소멸자는 서브 클래스의 소멸자 실행이 끝날 때 자동으로 호출됩니다.
- 슈퍼 클래스의 소멸자는 서브 클래스가 소멸자를 제공하지 않더라도 항상 호출됩니다.

## 소멸자 예시

```swift
class A {
    var x = 0
    var y = 0

    deinit {
        print("인스턴스가 메모리에서 해제됩니다.")
    }
}

var a: A? = A()

a = nil     // 인스턴스가 메모리에서 해제됨 -> 소멸자 호출
```
