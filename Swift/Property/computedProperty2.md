# 계산 속성 이해하기

계산 속성은 초기화된 다른 저장 속성의 값을 사용해서 계산한 결과값을 갖는 속성의 형태를 가진 메서드입니다. 그렇다면 메서드가 아닌 속성의 형태로 구현하는 이유는 무엇일까요?

- 계산 속성의 값을 읽고 설정하는 두 가지 메서드를 한곳에 한번에 구현할 수 있다.
- 메서드를 호출하는 번거로운 과정 없이 일반적인 속성을 사용하는 방식으로 값을 읽거나 설정할 수 있다.
- 따라서 편리하고 명확한 방식이다.
- 계산 속성은 메서드를 읽기 쉽고 사용하기 편한 형태인 속성으로 변환해놓은 것이다.

결국 계산 속성을 사용하는 이유는 개발자들이 읽기 쉽고 사용하기 편한 코드를 작성하기 위함입니다.

## 주의점

1. 계산 속성은 속성의 형태를 가진 메서드입니다.

   - 인스턴스 초기화 시 별도의 메모리 공간을 가지지 않습니다.
   - 값을 읽거나 설정할 때 다른 저장 속성에 접근해서 계산한 결과값을 리턴하거나 세팅합니다.
   - 값을 읽을 때 데이터 영역에 저장된 get() 메서드 저장 주소를 참조해서 실행합니다.
   - 값을 설정할 때 데이터 영역에 저장된 set() 메서드 저장 주소를 참조해서 실행합니다.

2. 항상 변하는 값이므로 변수(var)로 선언해야 합니다.

3. 반드시 자료형을 선언해야 합니다.

   - 메서드이기 때문에 리턴타입이 필요하다고 생각하면 됩니다.

4. 값을 읽는 get은 반드시 선언해야 합니다.

   - 값을 읽는 것은 필수이며 값을 설정하는 set은 선택입니다.
