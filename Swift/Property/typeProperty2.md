# 타입 속성 이해하기

## 저장 타입 속성 이해하기

먼저 저장 타입 속성에 대해 알아봅니다.

1. 저장 타입 속성은 타입 자체에 속합니다.

   - 타입 자체에 속한다는 의미는 타입으로 생성한 인스턴스에 속하지 않는다는 것입니다.
   - 타입 자체에 속하므로 저장 타입 속성은 메모리 데이터 영역의 타입 정의 내부에 위치합니다.
   - 저장 타입 속성은 초기화되면 타입 정의 내부에서 메모리 공간을 갖습니다.

2. 저장 타입 속성은 반드시 기본값이 필요합니다.

   - 일반적인 저장 속성은 인스턴스를 생성할 때 생성자로 속성을 초기화합니다.
   - 하지만 저장 타입 속성은 별도의 생성자가 존재하지 않습니다.
   - 따라서 저장 타입 속성을 초기화하고 사용하려면 반드시 기본값을 가지고 있어야 합니다.
   - 저장 타입 속성을 선언할 때 기본값을 할당하면 됩니다.

3. 자체적으로 지연(lazy) 속성의 성격을 갖습니다.

   - 저장 타입 속성은 지연 속성의 성격을 가지기 때문에 해당 속성에 접근했을 때 기본값으로 초기화됩니다.
   - 지연 속성의 성격을 갖는 것뿐, lazy 키워드로 선언할 필요는 없습니다.

4. 특정 인스턴스에 속한 것이 아니기 때문에 인스턴스 이름으로 타입 속성에 접근할 수 없습니다.

   - 타입 자체에 속하기 때문에 타입 이름으로 타입 속성에 접근할 수 있습니다.

5. 저장 타입 속성은 변수(var) 또는 상수(let)로 선언할 수 있습니다.
