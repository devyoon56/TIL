# 프로토콜 지향 프로그래밍(Protocol Oriented Programming)

상속의 단점과 프로토콜 지향 프로그래밍에 대해 알아봅니다.

## 상속의 단점

상속의 단점은 다음과 같습니다.

1. 다중 상속이 불가능하다.(스위프트)
2. 슈퍼 클래스의 기본적인 메모리 구조를 반드시 따른다. 즉 필요없는 속성과 메서드도 상속한다.
3. 클래스만 상속 가능하다.

## 프로토콜 지향 프로그래밍

프로토콜 지향 프로그래밍의 특징은 다음과 같습니다.

1. 여러개의 프로토콜을 채택할 수 있다. 즉 다중 상속과 유사하다.
2. 메모리 구조에 대한 요구사항이 없다. 선택적으로 멤버를 구현할 수도 있다는 의미이다.
3. 모든 타입이 프로토콜을 채택할 수 있다.
4. 프로토콜의 확장에서 프로토콜 요구사항 메서드의 기본 구현을 제공한다.
5. 프로토콜의 확장을 적용받는 타입을 제한할 수 있다.
