# 변수와 상수

## 변수와 상수는 왜 필요할까?

변수와 상수는 프로그램에서 사용되는 데이터를 메모리에 임시적으로 저장하기 위해 사용됩니다. 변수와 상수를 선언하면 메모리의 주소 공간을 쉽게 인식하고 사용할 수 있습니다. 메모리에 할당된 주소 공간을 외우지 않아도 되기 때문에 코드를 작성할 때 단순하게 변수와 상수명만 참조하면 됩니다.

## 변수와 상수

- 상수(constant)는 값을 한 번 설정하면 바꿀 수 없지만
- 변수(variable)는 미래에 다른 값으로 바꿀 수 있습니다.
- 상수는 바꿀 수 없다는 의미로 immutable,
- 변수는 바꿀 수 있다는 의미로 mutable로 표현하기도 합니다.

## 변수와 상수 선언하기1

- 상수는 **let**으로, 변수는 **var**로 선언합니다.

```swift
let name: String = "Jake"   // type annotation
var age = 29        // type inference
var job: String     // 변수에 값을 할당하지 않고 선언

age = 30            // 변수는 값 변경 가능
name = "제이크"     // 상수는 값 변경 불가능, 에러 발생
job = "iOS 개발자"  // 변수에 값을 할당
age = "삼십"        // 변수 타입이 정해지면 다른 타입의 데이터 할당 불가능(type safety)
```

## 변수와 상수 선언하기2

쉼표를 사용하면 여러 개의 상수나 변수를 한 줄로 선언할 수 있습니다.

```swift
let scoreA = 100, scoreB = 90, scoreC= 80, scoreD = 70
var myScore = 100, myGrades = "A+"  // 다른 타입을 섞을 수도 있습니다.
```

아래와 같은 코드는 에러가 발생합니다.

```swift
var x, y, z = 0.0, 0.1, 0.2     // 에러 발생
```

## 변수 대신 상수를 사용하는 이유

코드에서 저장 값을 바꾸지 않는다면, 항상 **let** 키워드의 상수로 선언합니다. 저장 값이 바뀌는 경우에만 변수를 사용합니다.

소스 코드의 가독성과 소스 코드 관리의 효율성 때문입니다.

[야곰님 블로그](https://blog.yagom.net/527/)

> 변수를 사용하지 않고 상수를 사용하는 이유는 여러가지가 있지만 가장 중요한 이유는 **가독성**입니다. 상수는 변하지 않는 값이기 때문에 상수로 값을 선언한다면 이후 코드에서 값의 변화가 없다는 사실을 주석이나 API 문서를 살펴보지 않고서도 직관적으로 알 수 있습니다. 즉, 차후 값의 변경을 신경쓰지 않아도 된다는 것입니다. 또 특정 값에 특별한 의미를 부여할 때 사용할 수도 있습니다. 예를 들어 입력받는 수의 최대 크기가 100이 라고 한다면, 100이라는 숫자에 의미를 부여하기 위해 **let maxInputValue = 100** 등으로 선언해두면 차후에 직관적으로 읽기도, 사용하기도 편리할 것입니다.
